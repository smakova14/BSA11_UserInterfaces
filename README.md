# BSA11_UserInterfaces

Проект по проектированию пользовательских интерфейсов для системы онлайн-записи барбершопа (BRB): формы представления продукта, экранные формы классификаторов и справочников, ключевые сценарии ролей, контроли ввода.

## 🛠 Навыки
- UI/UX Design (wireframes)
- Screen Flow Design
- Data Validation & Error Handling
- User Role Scenarios
- Accessibility & Localization

## 📋 Описание проекта
В ходе работы я разработала пользовательские интерфейсы для трёх ролей: менеджер, клиент, мастер. Описаны форм-факторы, локализация, экранные формы справочников, ключевые сценарии, контроли ввода и логирование.

### Что именно было сделано:
1. **Форма представления продукта (ex00)**: Определила устройства (Web: desktop + mobile), тип продукта (гибридный), способы управления (мышь/клавиатура + сенсор), локализацию (русский язык, российские форматы).
2. **Экранные формы классификаторов и справочников (ex01)**: Составила список справочников (Сотрудники, Роли, Услуги, Клиенты, Слоты). Разработала макеты форм списка и операций, описала основные и обеспечивающие операции (сортировка, фильтрация, поиск, логирование).
3. **Данные на экранных формах (ex02)**: Для каждого справочника описала поля (краткое наименование, тип, длина, обязательность, проверки, сообщения об ошибках). Указала потребность в пагинации и расчёте сумм. Перечислила возможные ошибки и их сообщения.
4. **Ключевые сценарии ролей (ex03)**: Разработала ключевые сценарии для менеджера, мастера и клиента. Построила схемы задач (блок-схемы) и сценарии экранных форм для основных процессов (регистрация клиента, бронирование услуги, просмотр расписания).
5. **Список экранных форм (ex04)**: Составила таблицу экранных форм с идентификаторами, основным назначением, другими действиями, необходимыми данными и правами доступа для каждой роли.
6. **Экранные формы – wireframe (ex05)**: Разработала детальные макеты форм (например, F-03 «Подтверждение записи», F-06 «Управление расписанием»). Для каждой формы описала основные и обеспечивающие операции, структуру данных (список и карточка), логирование.
7. **Описание контролей (ex06)**: Для ключевых форм (F-04 регистрация, F-06 добавление слота, F-03 подтверждение записи) описала контроли валидации: проверки формата, уникальности, доступности слота, сообщения об ошибках, результат при негативном/позитивном завершении, логирование.
